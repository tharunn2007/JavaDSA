HOW JAVA CODE EXECUTES?
    .java file(human readable source code) ---compiler (entire file)--->.class file( byte code,this code will not directly run on a system and need JVM to run this,reason why Java is platform independent )---interpreter(line by line)--->Machine binary code

    unlike cpp,c which compiles the source code that directly convert to machine code in the form of .exe file which is platform dependent but java converts into a intermediate language of java that can be run only with JVM so if we download JVM in any OS be it linux,mac or windows we can take the java code and run it with no compatibility issues.

MORE ABOUT PLATFORM INDEPENDENCE IN JAVA AND HOW IT IS DONE?
    Means byte code can run on all OS
    Normally compiler convert to machine code directly to get a .exe file which is platform dependent( they are compiled into native machine binary code and are structured according to OS specific binary formats that only the OS can read)
    Java as a language itself is platform independent but JVM is platform dependent as it is tailored to specific OS that converts to byte to native machine code
